一、需求分析
========================================================================

用户需求:
- 创建一个 HTML 坦克大战游戏
- 上传到 GitHub 变成一个仓库

设计讨论:
1. 游戏风格: 经典 FC 坦克大战 (复古像素风格)
2. 游戏模式: 单人模式 (玩家 vs AI 敌人)
3. GitHub 上传: 用户手动上传

========================================================================
二、技术方案
========================================================================

选择方案: 单 HTML 文件 + Canvas 渲染

方案优点:
- 所有代码在一个 index.html 里
- 使用 HTML5 Canvas 渲染，像素风格完美还原
- 双击即可运行，无需服务器
- 适合 GitHub Pages 直接部署

方案缺点:
- 代码都在一个文件里，维护性一般

========================================================================
三、游戏设计
========================================================================

核心玩法:
- 地图: 13x13 的网格系统
- 玩家坦克: 方向键移动，空格射击
- 敌方坦克: AI 控制，从地图顶部生成
- 基地: 地图底部中央的鹰标，被摧毁则游戏结束
- 目标: 消灭所有敌方坦克过关

地形元素:
+--------+------------------------------------------+
| 类型   | 说明                                     |
+--------+------------------------------------------+
| 砖墙   | 可被子弹摧毁                             |
| 钢墙   | 不可摧毁                                 |
| 河流   | 坦克不可通过，子弹可飞越                 |
| 草地   | 坦克可通行，覆盖坦克（隐藏效果）         |
| 冰面   | 坦克可通行                               |
+--------+------------------------------------------+

道具系统:
+--------+------------------------------------------+
| 道具   | 效果                                     |
+--------+------------------------------------------+
| 星星   | 升级坦克（射速/速度提升）               |
| 炸弹   | 消灭当前所有敌方坦克                     |
| 护盾   | 临时无敌                                 |
| 铲子   | 临时加固基地周围为钢墙                   |
+--------+------------------------------------------+

敌方坦克类型:
+--------+------------------------------------------+
| 类型   | 特点                                     |
+--------+------------------------------------------+
| 普通   | 标准速度和生命值                         |
| 快速   | 移动速度快                               |
| 重装   | 生命值高（3 点血）                       |
+--------+------------------------------------------+

关卡设计:
- 预设 5 个关卡，不同地图布局
- 每关 20 个敌方坦克
- 通关后自动进入下一关

========================================================================
四、技术实现细节
========================================================================

4.1 渲染系统
- 使用 Canvas 2D API
- 16x16 像素块绘制
- 60fps 游戏循环
- image-rendering: pixelated 保持像素清晰

4.2 地图系统
- 13x13 网格，每格 32x32 像素
- 总画布尺寸: 416x416 像素
- 地形类型用数字编码 (0-7)

4.3 坦克系统
- 玩家坦克: 键盘控制 (方向键/WASD)
- 敌方坦克: AI 控制，随机改变方向
- 碰撞检测: 基于网格的碰撞系统

4.4 子弹系统
- 子弹速度: 玩家 4，敌人 3
- 子弹可摧毁砖墙
- 子弹碰钢墙消失
- 子弹可飞越水面
- 子弹 vs 子弹互相抵消

4.5 AI 系统
- 敌方坦克随机改变方向
- 碰到障碍物立即换方向
- 随机射击间隔

4.6 道具系统
- 击败敌人 20% 概率掉落道具
- 道具闪烁显示
- 玩家触碰即生效

4.7 爆炸效果
- 粒子系统
- 不同大小的爆炸动画
- 颜色渐变效果

   - loadLevel() - 加载关卡

5. 更新逻辑 (约 200 行)

   - updateEnemies() - 敌人更新
   - updateBullets() - 子弹更新
   - spawnEnemies() - 生成敌人
   - checkCollisions() - 碰撞检测

6. 渲染逻辑 (约 200 行)
   - render() - 主渲染函数
   - drawTile() - 绘制地形
   - drawTank() - 绘制坦克
   - drawBullet() - 绘制子弹
   - drawExplosion() - 绘制爆炸

7. 辅助函数 (约 50 行)
   - canMove() - 移动检测
   - rectsOverlap() - 矩形碰撞
   - addExplosion() - 添加爆炸效果
   - updateUI() - 更新界面

十一、Bug 修复记录
========================================================================

Bug #1: 坦克卡脚问题
------------------------------------------------------------------------
问题描述:
- 坦克在移动过程中会卡在墙角或障碍物边缘
- 当坦克稍微偏离网格线时，无法顺畅地沿着墙壁移动
- 改变方向后坦克仍卡在原位

问题原因:
- 移动系统使用浮点数速度（1.5 像素/帧）
- 碰撞检测基于网格系统
- 缺少网格对齐机制，导致坦克位置与网格不对齐

解决方案:
1. 添加 snapToGrid 函数，将坐标对齐到最近的网格线
2. 添加 snapTankToGrid 函数，处理坦克方向改变时的对齐
3. 在玩家和敌方坦克改变方向时自动调用对齐函数
4. 当移动受阻时，尝试对齐到网格线后继续移动

技术实现:
- 对齐容差设置为 8 像素
- 垂直移动时对齐 X 轴
- 水平移动时对齐 Y 轴
- 对齐后重新检测碰撞，确保移动有效

修复效果:
- 坦克可以顺畅地沿着墙壁移动
- 方向切换更加流畅
- 消除了卡脚现象
